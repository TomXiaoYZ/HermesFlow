## 6. 数据采集服务 (Rust) ⭐ **技术栈变更**

### 6.1 服务概述
**技术栈**: Rust + Tokio + Actix-web  
**端口**: 18001 (数据采集), 18002 (数据处理)  
**状态**: 📋 规划中（旧版Python已归档，新版采用Rust重写）

**技术栈变更说明**：
- 原Python实现 → 改为Rust实现
- **核心优势**：μs级延迟、零GC暂停、强类型安全、高并发性能
- **性能目标**：延迟<1ms (P99)、吞吐量>100k msg/s

## 📝 开发规范

### 代码提交规范
```
[module:模块名] type: 描述

示例：
[module:strategy-engine] feat: 新增策略模板功能
[module:user-management] fix: 修复登录超时问题
[module:data-engine] perf: 优化数据采集性能
```

### 分支管理
- `main`: 生产环境分支
- `dev`: 开发环境分支
- `feature/*`: 功能开发分支
- `hotfix/*`: 紧急修复分支

### 进度更新流程
1. 开发功能并完成自测
2. 编写并运行测试用例
3. 在本地Docker环境验证
4. 更新本文档（progress.md）状态为"🔄 待用户验证"
5. 通知用户进行验证
6. 用户验证通过后，更新状态为"✅ 已完成"
7. 提交代码到Git仓库
