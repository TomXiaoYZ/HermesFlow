#### 3.2.4 CI/CD配置

##### GitHub Actions工作流

GitHub Actions工作流配置文件位于 `.github/workflows/deploy.yml`，主要功能如下：

```yaml
name: Deploy to AWS

on:
  push:
    branches: [ main, dev ]

env:
  AWS_REGION: ap-southeast-1
  EKS_CLUSTER_NAME: hermesflow
  ECR_REPOSITORY: hermesflow

jobs:
  deploy:
    name: Build and Deploy
    runs-on: ubuntu-latest
    steps:
    - 检出代码
    - 配置AWS凭证
    - 登录ECR
    - 设置环境后缀
    - 构建和推送镜像
    - 更新kubeconfig
    - 部署到EKS
    - 验证部署
    - 运行集成测试
    - 发送部署通知
```

##### 环境配置要求

1. GitHub Secrets配置
   - `AWS_ACCESS_KEY_ID`: AWS访问密钥ID
   - `AWS_SECRET_ACCESS_KEY`: AWS访问密钥
   - `SLACK_WEBHOOK_URL`: Slack通知webhook地址

2. AWS资源准备
   - ECR仓库创建
   - EKS集群配置
   - IAM角色和策略设置

3. Kubernetes配置
   - 开发环境配置目录：`infrastructure/kubernetes/dev/`
   - 生产环境配置目录：`infrastructure/kubernetes/prod/`

##### 自动化流程

1. 分支触发规则
   - `main`分支推送触发生产环境部署
   - `dev`分支推送触发开发环境部署

2. 自动化步骤
   - 自动构建和推送Docker镜像
   - 自动更新Kubernetes配置
   - 自动运行集成测试（dev环境）
   - 自动发送部署状态通知
